# 创建新文档工作流

此工作流用于根据模板快速创建结构化文档。

## 使用方法

```
/create-doc
```

## 工作流步骤

### 1. 询问文档类型
询问用户需要创建什么类型的文档：
- 技术规格文档 (`docs/specs/`)
- 指南或教程 (`docs/guides/`)
- 文章 (`content/articles/`)
- 故事 (`content/stories/`)
- 笔记 (`notes/`)

### 2. 获取文档信息
询问以下信息：
- 文档标题
- 简短描述
- 目标受众（技术人员/普通读者）
- 是否需要使用模板

### 3. 加载模板（可选）
如果用户选择使用模板：
- 检查 `templates/` 目录中的可用模板
- 加载相应模板
- 准备模板变量替换

### 4. 创建文档结构
根据文档类型创建基本结构：

**技术规格文档结构**：
```markdown
# [标题]

## 概述
[简短描述]

## 背景
[问题背景]

## 目标
[要达成的目标]

## 设计方案
[详细设计]

## 实现细节
[具体实现]

## 验证方法
[如何验证]
```

**文章结构**：
```markdown
# [标题]

> [引言或摘要]

## 正文
[内容]

## 总结
[总结]
```

### 5. 填充内容
- 根据用户提供的信息填充文档
- 如果使用模板，执行变量替换
- 保持 Markdown 格式规范

### 6. 保存文档
- 根据文档类型保存到相应目录
- 文件命名使用小写和下划线（例如：`my_document.md`）
- 确认保存成功

### 7. 更新索引（可选）
如果需要，更新 `README.md` 中的文档索引：
```markdown
## 文档索引
- [文档标题](path/to/document.md) - 简短描述
```

## 示例

**用户输入**：
```
请帮我创建一个关于用户认证系统的技术规格文档
```

**工作流执行**：
1. 识别类型：技术规格文档
2. 询问详细信息：标题、目标受众等
3. 创建文档结构
4. 保存到 `docs/specs/user_authentication_spec.md`
5. 更新 README.md 索引

## 注意事项

- ✅ 仅创建 `.md` 或 `.txt` 文件
- ✅ 遵循项目目录结构
- ❌ 不创建任何可执行文件
- ❌ 不在文档中嵌入可执行代码
